# 功能任务列表

> 面向 Telegram 管理机器人后续迭代的任务清单。

## P0（优先落地）

- [ ] 入群验证流程完善  
  包含验证题库（随机题/按钮题）、失败重试次数、白名单豁免、验证审计日志。

- [ ] AI 欢迎语策略  
  支持多模板轮换、按时间段欢迎语、按群类型差异化语气、失败回退模板优化。

- [ ] 反刷屏与反 Raid  
  检测短时入群激增、重复文案洪峰、相似昵称批量入群，自动切换高压策略。

- [ ] 自动巡检与告警  
  定时检查权限缺失、Webhook 异常、AI 超时率、动作失败率并主动告警。

## P1（治理能力增强）

- [ ] 风险用户画像  
  输出近 7/30 天风险分、违规类别分布、活跃时段、历史处置轨迹。

- [ ] 策略模板系统  
  提供 strict/balanced/relaxed 之外的场景模板（交易群/客服群/闲聊群）并支持一键切换。

- [ ] 管理动作审批流  
  对封禁、提权、改群资料等高风险动作增加双人确认与操作留痕。

- [ ] 申诉工单闭环  
  申诉状态机（待处理/通过/驳回）、处理 SLA、通知回执、可追溯审计链。

## P2（平台化与运营）

- [ ] 运营报表自动化  
  生成周报/月报：违规趋势、误杀率、回滚率、规则命中 Top、高风险用户 Top。

- [ ] 多群统一治理面板  
  跨群对比指标、批量下发策略、异常群优先级排序、集中处置入口。

- [ ] 全量成员沉淀方案  
  基于入群事件 + 发言事件构建成员池，补齐“成员全量近似视图”与生命周期状态。

- [ ] 可观测性与稳定性  
  增加关键链路指标（验证成功率、欢迎发送成功率、AI 失败率）与错误追踪。
